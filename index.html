# House Sitter Happy Hour — Website This is a simple single‑page site for managing house‑sitting signups with friends and family. ## 🚀 How to publish with GitHub Pages 1. **Create a GitHub repository** Log in to GitHub and click **New Repository**. Name it something like `housesitter-site`. 2. **Upload files** - Upload `index.html` (this file). - Upload your image: `HH-pano.jpg`. 3. **Enable GitHub Pages** - Go to your repo → **Settings → Pages**. - Under *Build and Deployment*, choose branch: `main` (or `master`), folder: `/root`. - Save. 4. **Get your live link** GitHub will give you a URL, usually like: `https://your-username.github.io/housesitter-site/` 5. **Share with friends & family!** 🎉 --- ## 📂 Example Project Structure Your repo should look like this: ``` housesitter-site/ ├── index.html # Main website file ├── HH-pano.jpg # Hero image shown at top of page └── README.md # Instructions (this file) ``` --- ## 🔄 How to update your site later If you need to make changes (e.g., new dates, perks, or design tweaks): 1. Edit your `index.html` file locally on your computer. 2. In GitHub, go to your repository and click **Add file → Upload files**. 3. Upload the updated `index.html` (it will replace the old one). 4. Commit the changes. Within a minute or so, GitHub Pages will automatically rebuild your site with the new version. Refresh the page at your live URL to see the updates. --- ## 📝 Notes - The form uses [Formspree](https://formspree.io/) to collect responses. You can change the `action` attribute inside `index.html` to your own Formspree endpoint. - A basic honeypot field is included to prevent spam. - You can later add Google reCAPTCHA for stronger protection once you have a domain. --- 💡 Tip: Keep `index.html` in the root of your repository for GitHub Pages to recognize it as your site entry point.
